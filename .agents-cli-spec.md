# Agent Spec: Workforce Knowledge Agent

## Overview

A three-agent pipeline that solves a real manufacturing problem: when experienced
engineers leave, their knowledge goes with them. This agent captures tacit expertise
through structured Q&A interviews, ingests existing documents, and surfaces the
right knowledge when the next person needs it.

Built as a Google ADK 2.0 multi-agent system deployed to Cloud Run (europe-west2).

## Example Use Cases

| Intent | Input | Expected output |
|--------|-------|-----------------|
| Retrieve | "What torque spec should I use for cylinder head bolts?" | Answer with source citation from KB |
| Capture | "I want to interview John Smith before he retires" | Structured Q&A interview, each answer stored |
| Ingest | "Here is our hydraulic press SOP: [text]" | Document chunked and stored in KB |
| Browse | "What knowledge do we have on CNC machining?" | List of relevant entries with sources |

## Agents

### Coordinator (root_agent)
Routes user intent to the correct sub-agent based on whether they want to
capture knowledge, ingest a document, or retrieve an answer.

### Capture Agent
Conducts a structured interview to extract tacit knowledge from an expert.
Asks one question at a time across categories: process steps, common failures,
safety considerations, tips from experience. Stores each answer via `store_knowledge`.

### Ingest Agent
Takes document text (SOPs, manuals, NCRs, emails) and stores it in the knowledge
base. Chunks logically, preserves source metadata.

### Retrieval Agent
Answers "how do we do X?" or "what causes Y?" using semantic search over the
knowledge base. Always cites the source of each answer.

## Tools

| Tool | Purpose |
|------|---------|
| `store_knowledge(content, source, category)` | Add an entry to ChromaDB |
| `search_knowledge(query, n_results)` | Semantic search over all entries |
| `list_sources()` | Show what knowledge exists and from whom |

## Architecture

- **Orchestration:** Google ADK 2.0, LLM delegation from root coordinator
- **Vector store:** ChromaDB (ephemeral for prototype; persistent for production)
- **LLM:** `gemini-flash-latest` via Vertex AI
- **Deployment:** Cloud Run, europe-west2
- **Pre-seeded:** 10 automotive manufacturing examples so the demo works immediately

## Constraints

- Never invent knowledge not in the KB; only answer from search results
- Always cite the source (expert name or document name) in retrieval answers
- Capture agent asks one question at a time, never dumps a list
- No external API calls; self-contained

## Success Criteria

- Retrieval agent finds relevant entries for manufacturing queries
- Source citations appear in every retrieval answer
- Capture agent stores at least one entry per interview turn
- Ingest agent stores document text and associates it with the given source name
- Demo works without any pre-ingestion step (seeded data)
